Description

This game was a part of my internship at Rajamuseo foundation. In the game, the player makes a passport and customizes it step by step and at the end prints it. From a printer. My role was as a lead coder. There was an another gamedesigner that also helped with coding the game.

Features

In the game, the player can input their name and customize the background, emblem, and picture on the passport. The player can take a selfie for the picture on the passport. The game runs on a chrome kiosk to prevent the players from tampering with it. The passport can be printed. The game looks weird in the video because it is designed for a specific computer that is in the museum.

Challenges

Main problems came from the fact that I had to have the game working on a kiosk while being able to print and have access to the camera to take a picture that could be put on the passport. My first approach was to try to make a UWP app which would run in the Windows 11 pro kiosk-mode, but the issue was that I could not get the printing working in kiosk mode. Printing needed to have a separate window for the selection, but kiosk mode would not allow that. I searched many approaches, but in the end it became clear that Windows kiosk mode would not work with the printing in Unity. The difficulty came down to the fact that Unity did not support the correct DLL, and I could not call windows to print a file of the passport. After getting stuck, I decided to search for a different approach. I ported the game to Unity WebGL. I redid the printing and picture taking since the WebGL is different compared to UWP app. I solved the problem with printing by calling the browser to print the page and adjusting the values manually on the browser to get it to print the page correctly. This approach was fine since the game will be installed on 4 laptops at the museum, and I was running out of time since the game needed to be released in the summer. I did the kiosk set up with chrome kiosk mode, which allowed the game to work and be reasonably secure. I also needed to rework the camera to take a picture of the player and put that picture to the passport in the game. I found out how I would have to start making my own JavaScript library to interact with the browser from Unity and had never done a code library for anything and had not coded anything in JavaScript. I decided to use Unity asset store component. I also had to find a way to host the file locally with https, so the browser would allow the camera to take pictures. I ended up finding an app called servez that did this.

Self reflection

I think I surpassed myself and became a much better developer. During the project, I faced my biggest problems until now. I was stuck on one of them for a month. I went through hundreds of different forum posts and ended up finding the solution that was right for me. There were many solutions that would have worked quite well for different games, but mine had a quite restrictive implementation that I had to follow. I feel like it is a miracle that I managed to get the game working, or maybe I am just lucky. During the project I had to learn many things about coding that I didn't think about before like DLL-files, compatibilities, group policies, commanding windows, file handling and limits of different game builds. I was left with a week of extra time, so the deadline came quite close.
